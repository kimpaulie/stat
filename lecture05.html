Lecture 05
The Normal Distribution

정규분포 — 세상의 많은 것이 종 모양으로 놓인다

사람의 키, 시험 점수, 다운로드 시간까지.
이름은 달라도 그림은 비슷해요. 가운데가 솟은 종 모양.
오늘은 이 아름다운 곡선의 비밀을 하나씩 풀어봐요.

PROLOGUE

여는 글 · MyTVLab의 고민

다운로드 시간 이야기

MyTVLab이라는 영상 서비스가 있어요. 통계팀이 다운로드 시간을 여러 번 재봤대요.

결과가 나왔어요. 평균 7초, 표준편차 2초. 그리고 하나 더. "이 시간은 정규분포를 따른다"는 거예요.

회사가 궁금한 건 세 가지. 9초 넘게 걸리는 영상은 몇 %인가? 99% 완료까지는 몇 초 걸리나? 기술을 개선하면 어떻게 달라지나?

오늘의 도구

이 세 질문에 답하려면 정규분포와 표준화를 알아야 해요. 오늘 수업이 끝나면 직접 계산해볼 수 있어요.

종 모양 곡선은 우연히 생긴 게 아니에요.
자연이 스스로 그려낸 모양이에요.
CHAPTER 01

연속 확률변수와 PDF

막대에서 곡선으로

4강에서는 이산 확률변수를 배웠어요. 주사위 눈금처럼 셀 수 있는 값들이었죠.

그런데 시간, 길이, 무게는 어때요? 7초, 7.1초, 7.13초… 중간값이 끝없이 많아요. 이런 걸 연속 확률변수라고 해요.

종이에 점을 찍는다고 생각해봐요.

이산은 "1, 2, 3, 4" 같은 눈금 위에만 점을 찍을 수 있어요.

연속은 눈금 사이 어디든 점을 찍을 수 있어요. 7초와 8초 사이에도, 7.5와 7.6 사이에도요.

Probability Density Function · 확률밀도함수 (PDF)
곡선으로 그려진 확률

연속 확률변수의 분포를 나타내는 함수예요. 기호는 f(x). 이 곡선의 아래 면적이 확률을 뜻해요.

PDF — 곡선 아래 면적 = 확률
연속변수의 확률 = 곡선 아래 색칠된 부분의 넓이.
중요한 점

연속변수에서는 정확히 한 점에 해당하는 확률은 0이에요. "정확히 7초" 확률은 0. 대신 "6.9초에서 7.1초 사이" 확률을 구해요. 이건 면적이니까 값이 나와요.

KEY TAKEAWAY
연속변수의 확률 = 곡선 아래 면적. 한 점은 0, 구간이 답.
CHAPTER 02

정규분포의 특성

통계학의 가장 유명한 곡선
Normal Distribution · 정규분포
N(μ, σ²)

평균 μ와 분산 σ²로 결정되는 종 모양의 연속 확률분포. 통계에서 가장 중요하고 가장 자주 등장하는 분포예요.

정규분포는 다음 네 가지 성질을 가져요.

  1. 종 모양. 가운데가 제일 높고 양옆으로 낮아져요.
  2. 좌우 대칭. 평균을 기준으로 완벽한 거울.
  3. 두 숫자로 결정. 평균 μ와 표준편차 σ만 알면 모양이 정해져요.
  4. 양 끝은 0에 가까워져요. 하지만 완전히 닿지는 않아요.
표준정규분포의 종 모양
가운데(평균 0)가 가장 높고, 좌우는 완벽하게 대칭.
f(x) = 1σ√(2π) · exp(−12 (x − μσ)²)

복잡해 보이죠? 겁먹지 않아도 돼요. 이 식은 종 모양을 그리는 설계도일 뿐이에요.

수식은 겁먹지 마세요

외울 필요 없어요. 중요한 건 μ와 σ 두 숫자만으로 곡선이 정해진다는 점이에요. 이 공식은 "종 그리는 법"이고, 우리는 그 결과 면적만 쓸 거예요.

확률 읽는 법

  • 전체 면적 = 1. 모든 확률의 합이니까 당연해요.
  • P(Z ≥ a) + P(Z < a) = 1. 두 쪽으로 나누면 합은 1.
  • P(Z < −a) = P(Z ≥ a). 좌우 대칭이니까 양쪽 꼬리는 같아요.
KEY TAKEAWAY
정규분포 = μ와 σ만으로 그려지는 좌우 대칭 종 곡선. 전체 면적은 1.
CHAPTER 03

μ와 σ의 역할

평균이 움직이면, 표준편차가 변하면

정규분포는 두 숫자로 결정된다고 했어요. 각 숫자가 하는 일을 따로 살펴봐요.

μ · 평균
곡선의 위치

μ가 커지면 곡선이 오른쪽으로 이동해요. 모양은 그대로, 위치만 달라져요.

σ · 표준편차
곡선의 퍼짐

σ가 작으면 뾰족하게 솟고, σ가 크면 넓고 낮게 퍼져요. 위치는 그대로, 모양이 달라져요.

평균이 바뀔 때 (σ는 고정)

Varying the mean · μ만 이동
세 곡선의 모양은 같아요. 위치만 바뀌었어요.

표준편차가 바뀔 때 (μ는 고정)

Varying the standard deviation · σ만 변화
σ가 작을수록 뾰족하게, 클수록 넓게 퍼져요.

비유해볼까요.

μ는 무대의 중심 위치. 마이크 스탠드가 왼쪽이냐 오른쪽이냐의 문제예요.

σ는 조명의 퍼짐. 스포트라이트는 좁고 강해요(σ 작음). 플러드라이트는 넓고 약해요(σ 큼).

KEY TAKEAWAY
μ = 위치 · σ = 퍼짐. 두 손잡이로 종 곡선을 자유자재로 조정.
CHAPTER 04

68-95-99.7 경험 법칙

기억하면 평생 쓰는 세 숫자

정규분포에는 유명한 법칙이 하나 있어요. 평균에서 몇 σ 떨어진 구간에 데이터가 얼마나 들어가는지에 관한 규칙이에요.

Empirical Rule · 경험 법칙
68-95-99.7의 마법

정규분포에서는:

  • 약 68%가 μ ± 1σ 안에
  • 약 95%가 μ ± 2σ 안에
  • 약 99.7%가 μ ± 3σ 안에
68-95-99.7 시각화
평균으로부터 1σ, 2σ, 3σ 구간의 면적이 곧 확률.

키로 비유해볼까요.

한국 성인 남성의 키가 평균 172cm, 표준편차 6cm인 정규분포라고 해봐요.

그러면 10명 중 약 7명은 166~178cm 사이(1σ 구간). 20명 중 19명은 160~184cm 사이(2σ 구간). 1000명 중 997명은 154~190cm 사이(3σ 구간)예요.

이 숫자를 외우면

계산기 없이도 어림셈이 가능해요. "우리 반 평균 70점, σ=10이면 90점 이상은 얼마?" → 2σ 밖 절반 = 2.5% 정도. 빠르죠.

KEY TAKEAWAY
68-95-99.7. 정규분포의 숨은 보물. 이 세 숫자가 빠른 감을 만들어요.
CHAPTER 05

표준정규분포

모든 정규분포의 기준점

세상엔 정규분포가 무수히 많아요.

키(μ=172, σ=6), 시험 점수(μ=70, σ=12), 다운로드 시간(μ=7, σ=2)… μ와 σ 조합에 따라 끝없이 달라져요.

매번 다른 곡선의 면적을 계산하긴 번거로워요. 그래서 기준 하나를 정했어요.

단위를 통일한다고 생각해봐요.

cm, inch, feet… 길이를 잴 때 단위가 다 다르면 헷갈리죠. 그래서 과학자들은 미터를 기준으로 정했어요.

정규분포도 마찬가지. 평균 0, 표준편차 1짜리 기준 곡선을 하나 정하고, 나머지는 모두 이걸로 환산해요.

Standard Normal Distribution · 표준정규분포
Z ~ N(0, 1²)

평균이 0이고 표준편차가 1인 정규분포. 보통 확률변수 기호는 Z를 써요. 모든 계산의 기준 곡선이에요.

Standard Normal N(0, 1²)
평균 0, 표준편차 1. 가장 높은 점은 z=0에서 약 0.4.

표준정규로 바꿔두면 정규분포표(Normal Table) 하나로 모든 확률을 찾을 수 있어요. 이 표는 다음 장에서 자세히 봐요.

우리가 풀 두 가지 문제

ⓐ z값이 주어지면 → 확률을 찾는다. 예: P(Z ≤ 0.73) = ?

ⓑ 확률이 주어지면 → z값을 찾는다. 예: P(Z ≤ z) = 0.73 일 때 z = ?

KEY TAKEAWAY
표준정규 Z ~ N(0, 1). 모든 정규분포를 이리로 데려와 풀어요.
CHAPTER 06

표준화 공식

어떤 정규분포든 Z로 번역하기

현실의 정규분포는 대부분 표준정규가 아니에요.

다운로드 시간은 μ=7, σ=2. 키는 μ=172, σ=6. 매번 곡선이 달라요.

그래서 이걸 표준정규로 번역하는 마법의 공식이 있어요. 줄 하나만 외우면 돼요.

Standardization · 표준화
핵심 공식

만약 X ~ N(μ, σ²)라면:

Z = X − μσ  ∼ N(0, 1²)

X에서 평균을 빼고, 표준편차로 나눠요. 그럼 Z가 나와요. Z는 항상 표준정규(평균 0, σ=1)를 따라요.

반대로 되돌리려면 이렇게.

X = μ + σZ

z값에 σ를 곱해서 원래 단위로 늘리고, 평균을 더해주면 X가 돼요. 표준화의 역연산이에요.

Z의 뜻을 직관으로 풀어봐요.

Z는 "X가 평균에서 몇 표준편차만큼 떨어졌나"를 재는 자예요. Z=2라면 평균보다 2σ 위. Z=−1이면 평균보다 1σ 아래.

키로 치면 172cm 평균에서 178cm인 사람은 Z = (178−172)/6 = 1. "1σ 위의 키"예요.

표준화 시각화 · X → Z 변환
X축 숫자만 바뀌고, 곡선 아래 면적(확률)은 같아요.
면적은 불변

X의 한 구간을 표준화하면 Z의 대응 구간이 돼요. 그 면적(확률)은 완전히 같아요. 눈금만 번역됐을 뿐이에요.

표준화 예시

X ~ N(7, 2²)에서 P(X > 9)를 구해봐요.

P(X > 9) = P(X − 72 > 9 − 72) = P(Z > 1)

X−7을 2로 나눈 게 Z예요. 부등식 양변에 같은 변환을 해줬을 뿐이에요. 확률 값은 그대로예요.

표준정규표에서 P(Z > 1) ≈ 0.1587. 즉 약 15.87%의 다운로드가 9초를 넘겨요.

KEY TAKEAWAY
Z = (X − μ)/σ. 이 한 줄이면 어떤 정규분포든 표준정규로 번역돼요.
CHAPTER 07

표준정규분포표(Z-table)는 P(Z ≤ z) 값을 미리 계산해둔 표예요. 보는 법만 익히면 한 번에 답이 나와요.

예시 ⓐ · P(Z ≤ 0.73) 찾기

  1. z = 0.73을 0.7 + 0.03으로 나눠요.
  2. 첫 열에서 0.7을 찾아 해당 행으로 가요.
  3. 첫 행에서 0.03을 찾아 해당 열로 내려가요.
  4. 교차점의 값을 읽어요 → 0.7673.
해석

P(Z ≤ 0.73) = 0.7673. 표준정규 변수가 0.73 이하일 확률이 약 76.73%라는 뜻. 곡선 아래 왼쪽 부분의 면적이에요.

P(Z ≤ 0.73) 의 시각화
왼쪽 색칠된 면적이 약 76.73%.
자주 쓰는 z값 암기

P(Z ≤ 1.00) ≈ 0.8413 · P(Z ≤ 1.64) ≈ 0.9495 · P(Z ≤ 1.96) ≈ 0.9750 · P(Z ≤ 2.33) ≈ 0.9901 · P(Z ≤ 2.58) ≈ 0.9951.

KEY TAKEAWAY
행(소수 첫째 자리) + 열(둘째 자리) = P(Z ≤ z). 이게 Z-table의 전부예요.
CHAPTER 08

역방향 — 확률에서 z로

표를 거꾸로 읽는 기술

이번엔 반대예요. 확률이 먼저 주어지고, 그에 맞는 z값을 찾아야 해요.

예: P(Z ≤ z) = 0.73 이 되는 z는?

  1. 표 내부에서 0.73에 가장 가까운 값을 찾아요.
  2. 그 값이 있는 행(소수 첫째 자리)과 열(둘째 자리)을 읽어요.
  3. 두 숫자를 더해요 → z 값이에요.
z0.000.010.020.03
0.50.69150.69500.69850.7019
0.60.72570.72910.73240.7357
0.70.75800.76110.76420.7673

표에서 0.73에 가장 가까운 값은 0.7291이에요. 이 값은 행 0.6과 열 0.01의 교차점에 있어요.

z ≈ 0.6 + 0.01 = 0.61
쓰는 상황

"상위 5%의 점수는 얼마?", "95% 확률로 도착할 시간은?" 같은 역문제에서 이 방법을 써요.

실전 · 다운로드 99% 완료 시간

X ~ N(7, 2²)에서 P(X ≤ x) = 0.99 인 x는?

  1. P(Z ≤ z) = 0.99 가 되는 z를 찾아요 → z ≈ 2.33.
  2. 표준화 역공식: X = μ + σZ = 7 + 2 × 2.33.
  3. x = 11.66초.

즉 약 11.66초가 지나면 99%의 다운로드가 끝나요.

KEY TAKEAWAY
내부 값 → 행+열 → z. 그리고 X = μ + σZ 로 원래 단위로 복귀.
CHAPTER 09

실전 예시 모음

오늘 배운 도구로 풀어보기

예시 1 · MyTVLab 다운로드 시간

X ~ N(7, 2²). 평균 7초, 표준편차 2초의 다운로드 시간.

Q1. 9초 초과 비율?

P(X > 9) = P(Z > 1) = 1 − 0.8413 = 0.1587. 약 15.9%의 다운로드가 9초를 넘겨요.

Q2. 99% 완료 시간?

z = 2.33 → x = 7 + 2 × 2.33 = 11.66초. 11.66초면 99%가 끝나요.

Q3. 기술 개선 효과는?

σ가 작아지면 곡선이 좁고 뾰족해져요. → 9초 초과 확률이 줄어들고, 99% 완료 시간도 짧아져요. μ가 작아지면 전체가 왼쪽으로 이동해요. 두 효과 모두 사용자가 체감하는 느림을 줄여요.

예시 2 · 호텔 객실 예약

배경: 동대문 호텔 객실 1000개. 설 연휴 취소 건수가 N(200, 50²)를 따른대요. 오버부킹 200건을 허용했어요.

Q. 점유율이 95% 미만일 확률?

취소가 너무 많으면 객실이 남아요. 95% 미만은 곧 취소가 250건을 넘는 경우.

P(취소 > 250) = P(Z > 250 − 20050) = P(Z > 1) ≈ 0.1587

즉 점유율이 95%에 못 미칠 확률이 약 15.9%예요. 드물게 객실이 남는 날이 있다는 뜻이죠.

예시 3 · 스키 파카 생산 계획

Sport Obermeyer의 파카 수요는 N(1200, 130²). 수요 충족 확률을 95%로 맞추려면?

  1. P(Z ≤ z) = 0.95 인 z ≈ 1.645.
  2. X = 1200 + 130 × 1.645 ≈ 1414벌.
  3. 즉 최소 1414벌을 생산하면 수요를 95% 확률로 감당해요.
95% 임계치 · 스키 파카
평균 1200 + 1.645σ ≈ 1414. 이 지점 아래 95%의 확률이 쌓여요.
KEY TAKEAWAY
모든 문제의 골격은 같아요. 표준화 → z로 바꾸기 → 표로 확률 읽기.
CHAPTER 10

다변량 정규분포 개요

여러 정규변수를 한 묶음으로

지금까지는 정규 확률변수 하나를 다뤘어요. 다운로드 시간, 키, 점수처럼요.

현실에서는 변수가 여럿이에요.

호텔 체인이면 두 지점의 손님 수. 포트폴리오면 여러 주식의 수익률. 공장이면 여러 라인의 생산량이죠.

Multivariate Normal · 다변량 정규분포
여러 정규변수의 묶음

두 개 이상의 정규 확률변수를 함께 다루는 분포예요. 각 변수의 평균 · 표준편차뿐 아니라 변수 사이의 상관관계까지 포함돼요.

오케스트라로 비유해볼까요.

한 악기 소리는 1차원 정규. 바이올린 하나의 음량 같은 거예요.

다변량 정규는 여러 악기가 동시에 연주될 때의 전체 소리예요. 각 악기의 세기, 그리고 악기들이 함께 커졌다 작아지는 정도까지 한 번에 담아요.

앞으로 풀 질문들

합의 분포는? 두 지점 합계 손님은 어떤 분포일까요.

차의 분포는? 동대문이 명동보다 손님이 많을 확률은요.

독립일 때와 아닐 때 뭐가 다를까요?

다음 세 챕터에서 하나씩 풀어봐요. 먼저 호텔 체인 예제로 상황을 잡고, 그다음 독립과 결합 정규 두 경우를 비교해요.

KEY TAKEAWAY
다변량 정규 = 여러 정규변수 + 상관관계. 한 묶음으로 보면 합도 차도 정규.
CHAPTER 11

호텔 체인 예제

동대문 + 명동, 두 지점을 묶어서

XYZ 호텔 체인이 있어요. 서울에 두 지점 — 동대문과 명동을 운영해요.

청소 스태프는 두 호텔이 함께 쓰는 풀로 200명. 관리자는 매번 이걸 몇 대 몇으로 나눌지 고민해요.

통계팀 보고에 따르면, 각 지점의 손님 수는 정규분포를 따른대요. 값은 아래와 같아요.

지점평균 μ표준편차 σ
동대문 (D)60080
명동 (M)72050

그리고 둘 사이엔 상관계수 ρdm = −0.65.

음의 상관이라는 건, 한 지점이 붐비면 다른 지점은 한가해지는 경향을 말해요. 서울 관광객이 같은 날 두 지점에 반반으로 쏠린다면 자연스러운 결과죠.

스태프 한 명이 방 6개 처리

각 스태프가 하루에 방 6개까지 처리한대요. 동대문에 90명, 명동에 110명을 배치했다고 해봐요.

동대문 · 90명 배치

처리 가능한 방: 90 × 6 = 540개. 인력 부족은 손님이 540 초과인 경우예요.

명동 · 110명 배치

처리 가능한 방: 110 × 6 = 660개. 인력 부족은 손님이 660 초과인 경우예요.

지점별 부족 확률 · 표준화로 풀기

동대문. D ~ N(600, 80²) 이니까:

P(D > 540) = P(Z > 540 − 60080) = P(Z > −0.75)

표준정규에서 P(Z > −0.75) ≈ 0.7734. 즉 동대문은 약 77% 확률로 인력 부족이에요. 사실상 거의 매일 바쁘다는 뜻.

명동. M ~ N(720, 50²) 이니까:

P(M > 660) = P(Z > 660 − 72050) = P(Z > −1.2)

P(Z > −1.2) ≈ 0.8849. 명동은 약 88% 확률로 부족이에요. 명동 쪽이 더 자주 모자라요.

왜 두 지점을 묶어서 봐야 할까?

스태프 풀이 공유잖아요. 한쪽이 남으면 다른 쪽을 도울 수 있어요.

그래서 "두 지점을 합쳐서" 부족 확률을 다시 계산해야 해요. 합계 D + M의 분포를 알아야 하죠.

두 지점을 합치면 · P(D + M > 1200)?

총 200명 × 6방 = 1200방이 하루 처리 한계예요. 부족 확률 = P(D + M > 1200).

여기서 질문이 나와요. D + M은 과연 정규분포일까요? 두 변수가 독립이 아니라 음의 상관이 있으니까, 답이 궁금해요.

다음 챕터에서

먼저 독립인 경우(ch12) 합이 정규임을 확인해요. 그다음 결합 정규(ch13)로 확장해서 상관까지 고려한 정확한 확률을 뽑아봐요.

KEY TAKEAWAY
동대문 + 명동을 묶어야 스태프 공유 효과가 보여요. 합의 분포부터 찾기.
CHAPTER 12

독립인 정규의 합

서로 영향 없을 때 가장 단순해요

가장 깔끔한 경우부터 살펴봐요. 두 정규 변수가 독립일 때예요.

독립 정규의 합 · Key Fact
합도 정규

X와 Y가 독립이고 각각 정규분포를 따르면, 임의의 상수 a, b에 대해 aX + bY도 정규분포예요.

정규 + 정규 = 정규. 모양이 무너지지 않아요. 이게 정규분포의 강력한 성질 중 하나예요.

평균과 분산 구하기

W = aX + bY 라고 둘게요.

E[W] = a · E[X] + b · E[Y]

평균은 그냥 가중합이에요. 쉬워요.

Var[W] = a² · Var[X] + b² · Var[Y]

독립이니까 공분산 항은 0이라서 사라져요. 분산도 가중합. 단, 계수는 제곱이 붙어요.

왜 a², b² 일까?

분산은 제곱 단위라서 그래요. 표준편차 σX를 a배 늘리면 aσX가 되고, 분산은 그걸 제곱한 a²σX²가 돼요.

예시 · 독립 호텔 체인을 가정해봐요

실제 호텔은 상관이 있지만, 만약 독립이라면 어떻게 되는지 먼저 연습. D ~ N(600, 80²), M ~ N(720, 50²), 독립 가정.

평균. E[D + M] = 600 + 720 = 1320.

분산. Var[D + M] = 80² + 50² = 6400 + 2500 = 8900.

표준편차. √8900 ≈ 94.3.

즉 독립 가정 하에서 D + M ~ N(1320, 94.3²). 이제 표준화로 부족 확률을 뽑을 수 있어요.

P(D + M > 1200) = P(Z > 1200 − 132094.3) = P(Z > −1.27)

P(Z > −1.27) ≈ 0.898. 독립이라면 약 90% 확률로 총합이 1200을 넘어요.

직관 체크

평균 1320이 용량 1200보다 커요. 그래서 초과 확률이 50%보다 훨씬 높은 게 자연스러워요. 독립 가정이 성립한다면 거의 매일 인력이 부족하다는 뜻.

KEY TAKEAWAY
독립이면 공분산 = 0. 평균은 더하고, 분산은 제곱 계수로 더하기만 하면 돼요.
CHAPTER 13

결합 정규의 합

상관이 있어도 괜찮아요

현실의 두 호텔은 독립이 아니에요. 상관이 있어요(ρ = −0.65). 그래도 합이 정규일까요?

Jointly Normal · 결합 정규
함께 묶어서 정규

X와 Y가 결합 정규분포(jointly normal)를 따르면, 임의의 상수 a, b에 대해:

  • a + bX 는 정규
  • X + Y 는 정규
  • aX + bY 는 정규

결합 정규면 어떤 선형결합도 정규예요. 독립이 아니어도 이 성질은 유지돼요.

분산에 공분산 항이 붙어요

평균 공식은 전과 같아요.

E[W] = a · E[X] + b · E[Y]

분산이 달라져요. 2ab · Cov[X, Y] 항이 추가돼요.

Var[W] = a²Var[X] + b²Var[Y] + 2ab · Cov[X, Y]

상관계수로 다시 쓰면 이렇게.

Var[W] = a²σX² + b²σY² + 2ab · σXσY · ρXY

공분산 Cov[X, Y] = σXσY · ρXY 를 대입한 것뿐이에요.

독립 공식은 특수한 경우

ρ = 0 이면 2ab · σXσY · 0 = 0. 그래서 독립 공식(ch12)이 자연히 나와요. 결합 정규 공식이 진짜 일반형이에요.

호텔 예제 · D + M 제대로 풀기

이제 ρdm = −0.65 를 반영해서 합의 분산을 구해봐요. a = b = 1.

Var[D + M] = 80² + 50² + 2·1·1·80·50·(−0.65)

하나씩 계산해요.

  1. 80² = 6400.
  2. 50² = 2500.
  3. 2 × 80 × 50 × (−0.65) = −5200.
  4. 합: 6400 + 2500 − 5200 = 3700.

표준편차는 √3700 ≈ 60.83. 즉 D + M ~ N(1320, 60.83²).

음의 상관이 준 선물

독립 가정에서는 σ ≈ 94.3 이었는데, 상관 −0.65가 들어가니까 60.83으로 줄었어요.

변동성이 줄었다는 뜻이에요. 한 지점이 많이 오면 다른 지점이 덜 오니까 합계의 흔들림이 작아져요. 리스크 분산 효과죠.

인력 부족 확률 재계산

P(D + M > 1200) = P(Z > 1200 − 132060.83) = P(Z > −1.97)

P(Z > −1.97) ≈ 0.976. 무려 97.6% 확률로 총합이 1200을 넘어요.

평균(1320)이 용량(1200)보다 이미 크고, σ도 작아서 거의 확정적으로 초과해요. 즉 스태프 풀 자체를 늘려야 하는 신호예요.

한 걸음 더 · 어느 쪽이 더 많을까?

질문: P(D > M) = ? 즉 동대문 손님이 명동보다 많을 확률이에요.

W = D − M 이라고 두고 결합 정규의 성질을 써요. a = 1, b = −1.

  1. E[W] = 600 − 720 = −120. 평균으로는 명동이 120명 더 많아요.
  2. Var[W] = 80² + (−1)²·50² + 2·1·(−1)·80·50·(−0.65) = 6400 + 2500 + 5200 = 14,100.
    음수 계수 × 음의 상관 = 양수. 차의 분산이 커졌어요.
  3. σW = √14,100 ≈ 118.7.
  4. P(W > 0) = P(Z > (0 − (−120))/118.7) = P(Z > 1.01) ≈ 0.156.

동대문이 명동보다 많을 확률은 약 15.6%예요. 평균이 낮으니 드문 일이지만, 상관 덕에 변동이 커서 가끔은 역전도 가능해요.

부호 주의!

차를 구할 때 b = −1이에요. 분산 공식에 그대로 넣어야 해요.

2ab에 (−1)이 들어가면 공분산 항의 부호가 뒤집혀요. 음의 상관이 차의 분산을 증가시키는 이유예요.

KEY TAKEAWAY
결합 정규 = 어떤 선형결합도 정규. 분산에 2·Cov(X, Y)만 더 챙기면 끝.
